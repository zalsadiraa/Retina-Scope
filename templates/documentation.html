Documentation Dataset

The Diabetic Retinopathy Severity Prediction application uses a dataset consisting of fundus images from patients diagnosed with diabetes. Each image is labeled to indicate the severity level of diabetic retinopathy. The prediction model aims to assist in identifying and classifying the severity of diabetic retinopathy based on the available images.

Visual Comparison: Normal vs. Diabetic Retinopathy

The following points illustrate the differences between a healthy retina and one affected by Diabetic Retinopathy:

  • Normal Retina: Shows clear blood vessels with no signs of damage.
  • Diabetic Retinopathy: Displays hemorrhages, aneurysms, and cotton wool spots, which indicate the severity of DR.

This comparison helps users understand the various indicators the model detects, such as hemorrhages, aneurysms, and hard exudates.

Dataset Overview

The dataset consists of fundus images taken from diabetic patients. These images are analyzed to detect symptoms of diabetic retinopathy, which causes damage to the retina and can lead to blindness if not treated. The dataset is used to train a machine learning model to classify the severity of diabetic retinopathy.

Data Fields

Each entry in the dataset consists of two main components: the fundus image and a label indicating the severity of diabetic retinopathy. Here is a description of the dataset attributes:

Attribute Description
id_code The fundus image taken from a diabetic patient.
diagnosis A label indicating the severity of diabetic retinopathy:
Severity Levels:
  • 0: No retinopathy (No DR)
  • 1: Mild diabetic retinopathy (Mild DR)
  • 2: Moderate diabetic retinopathy (Moderate DR)
  • 3: Severe diabetic retinopathy (Severe DR)
  • 4: Proliferative diabetic retinopathy (Proliferative DR)

Data Split Explanation

The dataset is divided into three main parts for training a machine learning model: the Training Set, Validation Set, and Test Set. Each set serves a distinct role during the training and evaluation process.

Training Set:

  • The training set is used to train the machine learning model. The model learns patterns, weights, and parameters from this data.

In the given code:

train_intermediate, val = train_test_split(df, test_size=0.15, stratify=df['type'])
  • 85% of the data from the original dataset (df) is assigned to train_intermediate.
  • The remaining 15% is allocated to the validation set.

Validation Set:

  • The validation set is used to evaluate the model during the training process. It helps with tuning hyperparameters, selecting the best model, and preventing overfitting.
  • 15% of the data from the original dataset is allocated to this set.

In the given code:

train_intermediate, val = train_test_split(df, test_size=0.15, stratify=df['type'])
  • The validation set is randomly sampled from the dataset while preserving the distribution of the target variable (type).
  • This split ensures that the validation set reflects the class distribution of the full dataset.

Test Set:

  • The test set is used to evaluate the final model after training. It is important that the test set is not used during training or hyperparameter tuning, as it should only provide an unbiased evaluation of the model’s performance.
  • 15% of the original dataset is set aside for the test set.

The code for splitting the training and test data is as follows:

train, test = train_test_split(train_intermediate, test_size=0.15 / (1 - 0.15), stratify=train_intermediate['type'])
  • Here, the train_intermediate set (which is 85% of the original data) is further split into:
    • 70% for the training set.
    • 15% for the test set.
  • This split ensures that the final test set is 15% of the original data, and that the class distribution in each of the subsets (train, validation, test) is proportional to the original dataset (stratify=df['type']).
